## Быстрый старт (macOS / Windows)

### 1. Создайте виртуальное окружение

**macOS / Linux:**
```bash
cd ktz_testing
python3 -m venv venv
source venv/bin/activate
```

**Windows (PowerShell):**
```powershell
cd ktz_testing
python -m venv venv
.\venv\Scripts\Activate.ps1
```

### 2. Установите зависимости

```bash
pip install -r requirements.txt
```

### 3. Настройте переменные окружения (опционально)

```bash
cp .env.example .env
# Откройте .env и укажите ANTHROPIC_API_KEY для ИИ-функций
```

Без API-ключа приложение работает, но ИИ-генерация использует заглушки.

### 4. Запустите приложение

```bash
python run.py
```

Откройте браузер: **http://127.0.0.1:5000**

## Функциональность

### Роли и доступ (RBAC)
- **Администратор**: полный доступ, управление пользователями, журнал
- **HR-специалист**: создание тестов, ИИ-генерация, аналитика
- **Руководитель**: просмотр результатов своего отдела, аналитика
- **Сотрудник**: прохождение тестов, личная история

### Модуль тестов
- 3 типа вопросов: один ответ, несколько ответов, открытый текст
- Настройка: время, проходной балл, количество попыток, перемешивание
- ИИ-генерация вопросов по теме (требует ANTHROPIC_API_KEY)

### Прохождение тестов
- Таймер с обратным отсчётом
- Автосохранение ответов в памяти
- Запрет Copy/Paste для текстовых ответов
- Предупреждение при попытке уйти со страницы


### Аналитика
- График активности за 30 дней
- Распределение баллов
- Средний балл по категориям и отделам
- Проблемные темы (низкая сдаваемость)
- Экспорт Excel

## База данных

SQLite-файл `ktz_testing.db` создаётся автоматически при первом запуске.  
Для сброса — удалите файл и перезапустите приложение.

**Модели:**
- `User` — сотрудники с ролями
- `Category` — категории тестов
- `Test` — тесты с настройками
- `Question` — вопросы (3 типа)
- `AnswerOption` — варианты ответов
- `TestAttempt` — попытки прохождения
- `UserAnswer` — ответы с ИИ-оценкой
- `AuditLog` — журнал всех действий
